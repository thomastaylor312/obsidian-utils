KEY INSIGHTS DISCOVERED
================================================================================

1. Three-Stage Pipeline (COMPOSABLE)
   Stage 1: reader → reads ALL files (no filtering)
   Stage 2: parser → filters .md files, creates AST
   Stage 3: frontmatter → extracts YAML from AST

2. Memory Management (EFFICIENT)
   - Arena allocation for all AST nodes
   - Iterator-based lazy evaluation
   - Lifetimes tied to arena (&'a AstNode<'a>)
   - Single deallocation on arena drop

3. Error Handling (GRACEFUL)
   - Individual file failures don't stop pipeline
   - Errors logged but continue processing
   - Partial results returned to application
   - Use of anyhow::Result<T> throughout

4. Markdown Parsing (FEATURE-RICH)
   - CommonMark + extensions via comrak
   - Supports: strikethrough, tables, footnotes
   - Wikilinks parsing enabled
   - YAML frontmatter delimiter support

5. Design Philosophy (PRAGMATIC)
   - Keep modules focused (142-172 lines each)
   - Compose stages into pipelines
   - No premature abstractions
   - Test each stage independently
